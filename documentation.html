Documentation

Website structure

The website have been developed using HTML for implementing the structure of the web pages, CSS for the visualization features and JavaScript for the interactive functions.

The main pages, each with its own HTML file, are Home, Issues, Documentation and Project. They all have in common the navigation bar and the footer, which have been taken from some predefined Bootstrap models and adapted to the project's purposes.

The section Home (index.html) contains a carousel giving access to the issues, an introduction about the magazine and its scope and a short presentation of the poets included in the issues.

The section Issues (issues.html) contains an index, which allows to reach the poets and the specific poems selected for the project, an introduction to explain the chosen issues and their subdivision, and finally the issues, in chronological order, each of which consists of a presentation of the poets and a grid to choose between the poems.

The section Documentation (documentation.html) contains information about the website structure, the texts (selection of poems and metadata) and the typographical styles.

The section Project (project.html) contains information about the purpose of the project and its team, and a disclaimer.

The navigation bar includes a final icon on the right, in order to allow the user to change theme on the fly, having different types of visualization of the whole website. For more information about the choice of the themes, see the paragraph Typographical styles. The footer, on the other hand, offers the possibility to reach the specific issues and the poems grouped by authors' names.

An additional HTML file (texts.html) has been created in order to visualize the poems and the related metadata. See the paragraph Texts metadata for a short analysis of the file "texts.html".
The typographical styles have been developed using a different CSS file for each theme.


Texts metadata

The web page texts.html has been developed to visualize the single poems and the related metadata. A Metadata Table is always visible on the left of the main section of the page, from which the user can select the poem and then select the metadata to highlight in the text.

During the choice of the poem, the user has to select an author's name and discover a list of buttons, representing the poems of the chosen author. If the user clicks on the title of a chosen poem, its text will appear to the right of the Metadata Table, in English language.

The whole website has been developed in English language, so all the poems are firstly displayed in English, but there's a button above the text, which proposes to the user to show the second language of the poem. The second language depends on the author, indeed Sappho's poems can also be visualized in Greek language, since it's the original language of the texts, while Kaur's and Dickinson's poems can be showed also in Italian language, since English is already the original language of the texts.

For what concerns the selection of the metadata, five buttons are constantly displayed, allowing the user to highlight the words or expressions in the texts which belong to the selected metadata class. In order to distinguish the classes and visually compare them on the text, each class has been associated with a color, filling also the button of the class' name, to make the association clear.

The five classes represent the recurrent themes which all the poems have, more or less, in common: Self-reference, intended as a reference to the author herself or to a group or community to which the author feels to belong; Body parts, which underlines the importance of the the human being as made of flesh and bones, beside soul and feelings, since the feelings have a strong influence on the body and viceversa; Generic places, both concrete and abstract; Negatives, including all the words and expressions with negative connotation, related for example to death, suffering, loneliness; Positives, including all the words and expressions with positive connotation, related for example to liveliness, happiness and healing.

The interactivity of the metadata section depends on different JavaScript functions:

  • functions to show the poems' buttons with the titles when click on the poet's name in the metadata table;
  • functions to show the poems on click of the title button, which are based on the .innerHTML method, in order to replace a specific part of the HTML file, in this case div containing the poems to the right of the Metadata Table. Basically, the text of the poem is inserted in the HTML section. The same method is used to display the poem in the second language;
  • functions to highlight words and expressions in the poems, related to the chosen classes, allowing to highlight all the classes contemporarily and to remove the highlight associated to a class when this latter is clicked again while active;
  • flags to control if the highlight functions are active, and if they are, they stay active even when other functions are called, such as the ones for showing and hiding the second language of the poems: a flag is set true when the corresponding highlighting function is called and set false when the function to remove the corresponding highlight is called.
